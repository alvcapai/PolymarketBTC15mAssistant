# [FUTURE FEATURE] Autopay Mullvad VPN via Bot Profits

## Overview
This feature aims to make the Polymarket trading infrastructure self-sustaining by automatically renewing the Mullvad VPN subscription using profits generated by the bots. This avoids manual monthly renewals and ensures 100% uptime for the Paraguay/Switzerland exit nodes required for Polymarket access.

## Architecture: The Bot Tax

1. **Trigger Condition:**
   - A monitoring script checks the Mullvad account expiration date weekly.
   - If expiration is < 7 days, the payment workflow is triggered.

2. **Funding (Withdrawal):**
   - The script checks the USDC balance on the Polymarket CLOB.
   - It performs an automated withdrawal of approximately 5.50 USDC (covering the 4.50 EUR monthly fee + swap slippage + network fees) from the CLOB to the Polygon mainnet wallet.

3. **Swap and Payment (Lightning Network):**
   - Step A: Generate a Lightning invoice via the Mullvad API.
   - Step B: Use a non-KYC swap provider (e.g., SideShift.ai or FixedFloat) via API to create a swap from USDC (Polygon) to BTC (Lightning).
   - Step C: Send the USDC to the swap provider. The provider then pays the Lightning invoice directly to Mullvad.

## Implementation Details (Tech Stack)
- **Monitoring:** Node.js script using scraping/internal API for Mullvad.
- **Transactions:** ethers.js for USDC transfers on Polygon.
- **Swaps:** REST API calls to SideShift/FixedFloat.
- **Wallet:** Uses the same PK as the trading bots.

## Security and Reliability
- **Non-Custodial:** No Lightning node is required on the server; swaps are executed atomically.
- **Notifications:** Integration with Discord via Claudio to report every successful renewal or balance issues.
- **Isolation:** Operates as a separate service (sidecar) to avoid interfering with the core trading logic.
